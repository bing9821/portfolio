NIUKEE E‑commerce Platform

One‑liner: Full‑stack shopping site with intelligent recommendations that auto‑apply bundle discounts for frequently co‑purchased items.

Role: Solo Six to Eight Months Stack: Node.js, Express, MongoDB, Handlebars GitHub

Problem → Outcome: Most stores show generic suggestions or hand‑made bundles. I implemented data‑driven Market Basket Analysis (Apriori) to surface real co‑purchase pairs, then applied discounts automatically at cart time—lifting average order value and smoothing the buyer journey.

  • Proof Points: Apriori rule mining (configurable support/confidence); secure REST APIs with bcrypt + structured errors; modular services (MBA, recommendations, email, shipping).
  • Results: Instant cart checks against 1,000+ rules; full flow catalog → checkout → tracking with emails; session auth with profiles, points, vouchers.
  • Skills: Backend Dev • Data Mining • REST API • DB Modeling • Auth & Security • Service Architecture • Full‑stack

Approach

Read details

Chose Apriori for explainable, auditable rules. Pre‑generated rules from historical transactions for speed. Used MongoDB for flexible product/order schemas. Separated concerns into services for maintainability and growth.

Architecture / Data Model

Read details
  • SSR: Handlebars templates with session data
  • Express routes → controllers → services (MBA/reco/email/shipping) → Mongoose models
  • Collections: User, Product, Cart, Order, Voucher, RedeemedVoucher (indexed)
  • Auth: express‑session + bcrypt; guarded routes
  • MBA: transactions → Apriori → rules store (CSV/collection) → real‑time cart match → discount calc
