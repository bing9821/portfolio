Treasure Hunt A* Solver

One‑liner: Hex‑grid pathfinding with complex state that optimizes routes through traps, rewards, and forced‑movement mechanics using custom heuristics.

Group project 3 weeeks Stack: Python, A*, Matplotlib, NumPy, Dataclasses GitHub

Problem → Outcome: Standard A* handles uniform‑cost grids. I built a stateful solver for odd‑q hex grids where room effects alter costs and movement. The system tracks energy multipliers, collected treasures, activated effects, and forced moves to compute truly optimal paths under compound penalties—solving 18‑step routes on 60‑cell boards in under 10 seconds.

Three Proof Points

  • Stateful A*: each node tracks position, treasures, activated effects, energy multiplier, and forced‑move queue
  • Hex odd‑q coordinates: correct 6‑direction movement with even/odd column offsets
  • Real‑time visualization: stepwise rendering with colored hexes and live f/g/h and open‑set views

Results

  • <10 s for 6×10 grids with 4 treasures and 13 traps/obstacles
  • Admissible heuristic blends Manhattan‑style distance, remaining treasures, and energy‑cost terms
  • 2,000+ node expansions with detailed logs: successor evaluation, trap avoidance, per‑step cost breakdowns

Skills: AI Search • Algorithm Design • Complex State Management • Graph Pathfinding • Visualization • Data Structures • Python OOP

Approach

Read details ▼

Used A* with a custom admissible heuristic; precomputed odd‑q neighbor tables for 6‑way movement; separated GameState (immutable), Node (priority item), and PathEvaluationInfo (logs); heap‑based open set (O(log n)), closed set keyed by full state signature; matplotlib animates progression with color‑coded effects.

Architecture / Data Flow

Read details ▼
  • Grid: odd‑q offset; neighbors vary by column parity (N, S, NE, NW, SE, SW)
  • GameState: position; collected_treasures; available_treasures; activated_effects; energy_multiplier; forced_steps_remaining; last_direction
  • Node: wraps GameState + g, h, f (f = g + h) for heap priority
  • Effects: Traps: T1 (2× energy), T2 (2× time), T3 (forced 2 steps), T4 (clear uncollected treasures) | Rewards: R1 (½× energy), R2 (½× time) | One‑time activation tracked in activated_effects
  • A* mechanics: min‑heap open set; closed set maps state signature → best g; generate successors → apply effects → enqueue
  • Heuristic: h = hex distance to goal + treasure penalty + energy‑cost estimate (tuned to remain admissible)
  • Output/vis: matplotlib RegularPolygon hexes; yellow path, green current; dynamic titles with step count and total cost

What I Learned & Next Steps

Read details ▼
  • Robust state hashing prevents collisions (include energy/effects/forced moves)
  • Odd‑q vs cube coords: neighbor quirks matter for correctness
  • Admissibility vs guidance: trap penalties in h risk overestimation—careful upper bounds are key
  • Python heapq details: tuple priors for deterministic tie‑breaks
  • Next: Apply T2 time multiplier to step costs (currently only T1 applied)
  • Explore bidirectional A* for larger boards
  • Partial‑order reduction to shrink state space (track only relevant effect activations)
  • Benchmark against Dijkstra, BFS, greedy best‑first
